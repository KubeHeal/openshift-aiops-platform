# ADR-004 Fix: InferenceService Image Mismatch

**Date:** 2026-03-03
**Status:** ✅ RESOLVED
**ADR:** 004 - KServe for Model Serving Infrastructure

---

## Problem Summary

**Issue:** predictive-analytics InferenceService in CrashLoopBackOff with 70+ restarts over 6 hours

**Error:**
```
ModuleNotFoundError: No module named 'xgboost'
```

**Root Cause:**
- The predictive-analytics model was trained using XGBoost
- The InferenceService was configured to use `kserve/sklearnserver:latest` image
- Standard sklearn runtime does NOT include XGBoost package
- Pod crashed on startup when trying to load XGBoost-trained model

**Secondary Issue:**
- scikit-learn version mismatch (model trained with 1.6.1, runtime has 1.5.2)
- Caused warnings but not fatal errors

---

## Investigation Steps

### 1. Checked InferenceService Status
```bash
oc get inferenceservice -n self-healing-platform
# Result: anomaly-detector READY, predictive-analytics NOT READY
```

### 2. Examined Pod Logs
```bash
oc logs predictive-analytics-predictor-5df864f697-jzvh4 -n self-healing-platform
# Found: ModuleNotFoundError: No module named 'xgboost'
```

### 3. Verified InferenceService Image
```bash
oc get inferenceservice predictive-analytics -o yaml
# Found: image: kserve/sklearnserver:latest (missing XGBoost)
```

### 4. Discovered Custom Image
```bash
oc get imagestream sklearn-xgboost-server -n self-healing-platform
# Found: image-registry.openshift-image-registry.svc:5000/self-healing-platform/sklearn-xgboost-server:latest
```

The custom image already existed with XGBoost support!

---

## Solution

### Fix Applied (Both SNO and HA Clusters):

```bash
oc patch inferenceservice predictive-analytics -n self-healing-platform --type='json' -p='[
  {
    "op": "replace",
    "path": "/spec/predictor/containers/0/image",
    "value": "image-registry.openshift-image-registry.svc:5000/self-healing-platform/sklearn-xgboost-server:latest"
  }
]'
```

### What Changed:
**Before:**
```yaml
spec:
  predictor:
    containers:
    - image: kserve/sklearnserver:latest  # ❌ No XGBoost
```

**After:**
```yaml
spec:
  predictor:
    containers:
    - image: image-registry.openshift-image-registry.svc:5000/self-healing-platform/sklearn-xgboost-server:latest  # ✅ Includes XGBoost
```

## Custom Image Details

**BuildConfig:** `sklearn-xgboost-server`
**Source:** `charts/hub/templates/sklearn-xgboost-buildconfig.yaml`

**Dockerfile:**
```dockerfile
FROM kserve/sklearnserver:latest
USER root
RUN pip install --no-cache-dir --target=/prod_venv/lib/python3.11/site-packages xgboost lightgbm
USER 1000
```

**Includes:**
- ✅ All sklearn server functionality
- ✅ XGBoost package
- ✅ LightGBM package

## Why Helm values.yaml Was Correct

**Checked values.yaml:**
```yaml
modelServing:
  sklearn:
    image: image-registry.openshift-image-registry.svc:5000/self-healing-platform/sklearn-xgboost-server:latest
```

**The Helm template was correct!** The deployed InferenceService must have been created before this configuration was set, or manually modified. The manual patch brings the cluster state back in sync with the intended Helm configuration.

## Lessons Learned

### ✅ What Worked:
1. Custom sklearn-xgboost-server image was already built
2. Helm values.yaml already had correct configuration
3. Simple patch fixed immediate issue

### ⚠️ To Improve:
1. **Version Mismatch:** Model trained with sklearn 1.6.1, runtime has 1.5.2
   - **Solution:** Update sklearn in custom image Dockerfile to match training version

2. **Model Training Pipeline:** Ensure training uses same dependency versions as serving runtime
   - **Solution:** Use same base image for training and serving

3. **Pre-deployment Validation:** Could have caught this before deployment
   - **Solution:** Add validation job that checks model dependencies match runtime
